## 코드 변경 안전 지침

모든 코드를 적용하거나 신규 기능을 추가할 때는 반드시 다음을 준수한다:

1. **기존 기능 보호**: 변경 전에 영향받는 파일과 기능을 파악하고, 기존 기능이 깨지지 않는지 반드시 확인한다.
2. **타입체크 필수**: 모든 변경 후 shared → backend → frontend 순서로 타입체크(tsc --noEmit)를 통과해야 한다.
3. **신중한 변경**: 한 번에 너무 많은 파일을 동시에 수정하지 않는다. 단계별로 변경하고 각 단계마다 검증한다.
4. **에러 제로 원칙**: 커밋 전에 에러가 하나도 없어야 한다. 에러가 있으면 반드시 수정 후 커밋한다.
5. **크로스 파일 일관성**: 백엔드 API 변경 시 프론트엔드 호출부도 함께 확인한다. import 경로, 함수 시그니처, 타입 정의가 모두 일치해야 한다.
6. **회귀 테스트**: 신규 기능 추가 시 관련된 기존 기능(같은 서비스, 같은 컴포넌트)이 정상 작동하는지 꼼꼼히 확인한다.
